☎️ Call Flow — Step by Step
1
Caller dials the Teams resource account phone number (PSTN Direct Routing).
2
ACS receives the call via Teams interop and fires IncomingCall to Function App.
3
Function answers call. ACS TTS plays: "Welcome to [Company]. Please say the name of the person you'd like to speak to."
4
ACS Speech Recognition listens. Caller speaks a name (e.g. "Sarah Jones").
5
Function queries Graph API: GET /groups/{id}/members — loads all staff display names + AAD Object IDs.
6
Fuzzy match (difflib / rapidfuzz) finds closest name. Confidence threshold checked (>70%).
7
TTS confirms: "Connecting you to Sarah Jones." — or "Sorry, I couldn't find that person. Transferring to reception."
8
Direct Routing transfer via transferCallToParticipant(MicrosoftTeamsUserIdentifier).
9
Call arrives in staff member's Teams client. Event logged to App Insights.

👥 Azure AD Security Group — How Staff Routing Works

Group name
VirtualReceptionist-Staff
Security group in Entra ID. Customer IT admin manages membership.
Adding a new staff member
Add them to the group
Azure AD → Groups → VirtualReceptionist-Staff → Members → Add. No code changes, no redeployment.
Removing a leaver
Remove from group
Remove from group membership. Bot will no longer match their name on the next call. Instant effect.
What Graph returns
displayName + id
Bot loads displayName for fuzzy matching and id (AAD Object ID) for the transfer target.
Fuzzy match threshold
> 70% confidence
Below threshold → transfer to default reception. Configurable in CONFIG block.
Graph API caching
5-minute in-memory cache
Group members cached per function instance to reduce Graph API calls on high call volumes.
